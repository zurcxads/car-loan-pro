## CRITICAL (Fix before launch)

### 1. Unauthenticated API Routes (19 routes)
Routes that accept requests without verifying the caller:
- `/api/admin/applications` — admin data exposed
- `/api/dashboard` — consumer data exposed
- `/api/dealers/[dealerId]/buyers` — dealer data exposed
- `/api/documents/download` — file download without auth
- `/api/documents/route` — document listing without auth
- `/api/documents/upload` — file upload without auth
- `/api/email/send` — could be abused to send spam
- `/api/lenders/decisions` — lender decisions exposed
- `/api/lenders/[lenderId]/applications` — lender app queue exposed
- `/api/notifications/unread-count` — minor but should be auth'd
- `/api/offers/[id]/select` — offer selection without auth
- `/api/plaid/create-link-token` — Plaid token generation exposed
- `/api/referrals` — referral data exposed
- `/api/results` — results without auth
- `/api/results/select` — result selection without auth
- `/api/vin/decode` — minor
- `/api/webhooks` — webhook management exposed
- `/api/webhooks/test` — test webhook trigger exposed
- `/api/dev/create-test-app` — should be gated to dev mode only

**Risk:** Anyone can call these endpoints directly. Admin/lender data leak, spam abuse, unauthorized actions.
**Fix:** Add auth middleware or `authenticateRequest()` check to each route.

### 2. recharts Still in Bundle
`recharts (^3.8.0)` is a heavy charting library (adds ~200KB+ to bundle). Decision was SVG-only charts in portals.
**Fix:** Remove from `package.json`, replace any remaining usage with inline SVG.

### 3. Open Redirect in Login Page
`src/app/login/page.tsx` line 22: `window.location.href = target` where `target` comes from `searchParams.get('redirect')`. An attacker could craft `/login?redirect=https://evil.com`.
**Fix:** Validate redirect is a relative path (starts with `/`) before redirecting.

## HIGH (Fix pre-launch)

### 4. 93 console.log/error Statements
Production code has 93 console statements. Most are `console.error` in catch blocks (acceptable for debugging) but 5 are `console.log` that leak info:
- `src/app/api/dev/create-test-app/route.ts:26` — logs test app data
- `src/app/lender/onboard/page.tsx:60` — logs form submission data
- `src/app/dealer/onboard/page.tsx:73` — logs form submission data
- `src/lib/application-events.ts:37` — logs application events
- `src/lib/setup-db.ts:24,33,202` — logs DB setup info

**Fix:** Replace `console.log` with proper error tracking. Keep `console.error` in catch blocks.

### 5. Demo Credentials Visible in Production
`src/lib/test-data.ts` and `src/app/dev/page.tsx` expose `AutoLoanPro2026!` password.
**Risk:** Low (these are demo accounts), but should be env-gated.
**Fix:** Only expose on dev mode, strip from production build.

### 6. npm Audit: 7 Vulnerabilities (3 low, 4 high)
**Fix:** Run `npm audit fix` and test. If breaking, address individually.

### 7. dangerouslySetInnerHTML Usage (4 instances)
- `src/app/resources/[slug]/page.tsx` — blog content rendering (3 instances)
- `src/app/admin/email-templates/page.tsx` — email preview

**Risk:** XSS if content is user-generated. Currently static content, so low risk.
**Fix:** Sanitize with DOMPurify before rendering, or switch to markdown parser.

- `robots.ts`, `stipulation-engine.ts`, `auth-types.ts`

**Fix:** Remove unused files. Some may be needed later — move to `_archive/` if unsure.

### 9. Large Bundle Chunks
- `9214-*.js`: 386KB (likely framer-motion + recharts)
- `549-*.js`: 204KB
- `fd9d1056-*.js`: 169KB

**Fix:** Remove recharts, lazy-load framer-motion on pages that use it, code-split portal pages.

### 10. CSP Has 'unsafe-inline' and 'unsafe-eval'
`script-src 'self' 'unsafe-inline' 'unsafe-eval'` — weakens XSS protection.
**Fix:** Use nonces for inline scripts, remove `unsafe-eval` (Next.js may need it in dev but not prod).

### 11. In-Memory Rate Limiter
Rate limiting uses in-memory Map — doesn't persist across serverless function invocations on Vercel.
**Fix:** Move to Vercel KV, Upstash Redis, or Supabase-based rate limiting for production.

### 12. Service Role Key Fallback
`src/lib/supabase.ts:40` falls back to anon key if service role isn't set. Could cause silent permission failures.
**Fix:** Throw error if service role key is missing in server contexts.

## LOW

### 13. Missing robots.ts Export
`src/app/robots.ts` exists but is flagged as unused — may not be properly exporting.
**Fix:** Verify robots.txt is served correctly at autoloanpro.co/robots.txt.

### 14. Next.js Dynamic Route Warnings
3 API routes trigger "couldn't be rendered statically" warnings during build. Not errors, but noisy.
**Fix:** Add `export const dynamic = 'force-dynamic'` to affected routes.

### 15. No CORS Configuration
No explicit CORS headers. Currently fine since all API calls are same-origin, but will matter when mobile app launches.

### 16. SSN Field Masking
SSN input uses `type="password"` with formatted display. Consider showing last 4 digits on review step.

### 17. Phone Formatting
Apply-v2 stores raw digits but formats display. Consider normalizing to E.164 format for Twilio integration.
